The Evolution of Artificial Intelligence & Machine Learning: An Interactive Timeline

Workshop 1 · Discussion 1.3 · Historical Research

Introduction

This artifact is an interactive, visually rich timeline that traces the evolution of AI and machine learning from Alan Turing's foundational 1950 paper through the current era of large language models and multimodal reasoning systems.

Description

The timeline covers 20 key milestones organized into color-coded eras: Early AI (1950s–1970s), AI Winters, Revival periods, the Deep Learning breakthrough era (2011–2017), and the LLM era (2020–present). Each entry includes the year, event name, detailed description, and key figures or organizations involved. The timeline features scroll-triggered animations, responsive design, and an alternating left-right layout for visual engagement.

Objective

To research and present the historical development of AI/ML in a format that is both educational and visually compelling, demonstrating the ability to synthesize complex technical history into an accessible resource.

Process

I studied three source types recommended by the course: a narrative video documentary on AI history, an IEEE scholarly article on AI evolution, and interactive data visualizations from Our World in Data. I then designed and coded a custom HTML/CSS/JavaScript timeline with era-based color coding, hover interactions, and intersection observer animations for a dynamic scroll experience.

Tools & Technologies Used

HTML5, CSS3 (gradients, animations, media queries), JavaScript (Intersection Observer API), Claude AI (research assistance), Google Fonts.

Value Proposition

Unique Value: This timeline is not a generic copy-paste exercise. It is a hand-crafted, interactive web resource that demonstrates both research depth and front-end development capability, skills that set apart a supply chain professional who can also build digital tools.

Relevance: Understanding where AI came from is essential for anyone leading its adoption. This resource can serve as a reference for colleagues, stakeholders, or team training sessions to contextualize why AI technologies work the way they do today.

FreightIQ: An AI-Powered Freight Management Chatbot

Workshop 1 · Assignment 1.4 · Hands-On AI Lab

Introduction

This artifact is a fully functional proof-of-concept chatbot, FreightIQ, that I designed and built to solve a real pain point in retail freight management: store leaders needing fast, conversational access to freight status, stocking priorities, and delay alerts without navigating multiple systems.

Demo

Description

FreightIQ is a Python-based agentic chatbot powered by Claude AI. It features a real-time chat interface where store leaders can ask natural-language questions about incoming freight, delivery ETAs, stocking priorities, and active delay alerts. The system uses a mock freight data server to simulate live data, and the Claude-powered agent server interprets user queries, calls the appropriate API tools, and returns structured, actionable responses. Quick-action chips allow one-tap access to common queries like "Next Delivery," "Stocking Priorities," "Delay Alerts," "All Trailers," and "Full Briefing."

Objective

To apply design thinking methodology and generative AI tools to develop a practical, working prototype that addresses a real operational challenge in retail freight management.

Process

I followed a design thinking process: empathize (identify store leaders' frustration with fragmented freight data across multiple systems), define (the need for a single conversational interface), ideate (sketch chatbot features and data model), prototype (build a full-stack application with a Python agent server, mock freight API, and responsive HTML/CSS/JS frontend), and test (verify responses across delivery queries, stocking priorities, delay alerts, trailer lookups, and full briefing scenarios with automated tests).

Tools & Technologies Used

Python, Claude AI (Anthropic API with tool use), Flask, HTML5, CSS3, JavaScript, Pytest, Design Thinking methodology.

Value Proposition

Unique Value: This artifact goes beyond theoretical understanding. It is a working, end-to-end AI application with a real agent architecture, API integration, and automated test coverage. The chatbot directly addresses a daily challenge I observe at Walmart, demonstrating practical AI application in supply chain.

Relevance: For any employer in retail or logistics, this demonstrates the ability to translate AI knowledge into tangible, deployable solutions. The combination of agentic AI design, full-stack development, and supply chain domain knowledge shows initiative and applied technical capability.

Workshop 3 · Discussion 3.4 · Communicating for Learning

Introduction

This artifact is an interactive, visual explainer that demystifies how neural networks process data, learn from mistakes, and improve over time. It uses a freight dock analogy — training a new associate through show, try, correct, repeat — to make the concept accessible to non-technical audiences.

Description

The explainer walks the reader through five steps: the freight dock analogy, neural network architecture (with an interactive canvas diagram), a live training simulator that shows loss decreasing and accuracy increasing over epochs, a shelf-classification demo where users click to see the network classify retail images, and a discussion of trade-offs including the black box problem. Each section includes scroll-tracked navigation and interactive elements.

Objective

To demonstrate the ability to communicate complex technical concepts — specifically how neural networks learn — in a way that is clear, engaging, and accessible to peers who may not have a deep technical background, while grounding the explanation in a real-world retail context.

Process

I started by writing a plain-language explanation of neural networks for the Discussion 3.4 assignment, using a freight dock analogy drawn from my daily work at Walmart. I then expanded this into a multi-section interactive web experience with a canvas-based network diagram, a training epoch simulator, and a clickable shelf classification demo. The design uses a dark theme with layered animations to keep the reader engaged through progressively deeper content.

Tools & Technologies Used

HTML5, CSS3 (custom properties, gradients, sticky navigation), JavaScript (Canvas API, Intersection Observer, DOM manipulation), Claude AI (content refinement), Google Fonts.

Value Proposition

Unique Value: This artifact goes beyond a static explanation. The interactive training simulator and clickable classification demo let the reader experience how a neural network learns, not just read about it. The freight dock analogy grounds abstract math in a tangible, relatable scenario that any operations professional can understand.

Relevance: For peers in the AIML program, this serves as both a learning resource and a template for how to communicate technical AI concepts to non-technical stakeholders — a critical skill for anyone leading AI adoption in their organization.

AI Bias in Healthcare: When Algorithms Inherit Inequality

Workshop 6 · Assignment 6.2 · AI Incidents & Ethics

Introduction

This artifact is an interactive case study that analyzes racial bias in a healthcare algorithm used to guide care decisions for approximately 200 million patients annually. Based on the landmark 2019 study by Obermeyer et al. published in Science, it explores how a seemingly neutral design choice — using healthcare spending as a proxy for health need — encoded structural racism into an AI system deployed at massive scale.

Description

The case study is organized into six interactive sections: a statistical overview of the incident with key metrics, a visual bias pipeline diagram showing how unequal access translated into algorithmic discrimination through a self-reinforcing feedback loop, a tabbed deep dive into three ethical issues (proxy variables, lack of demographic auditing, and scale amplification) each with analysis and proposed solutions, a discussion of how COVID-19 amplified these biases during triage and resource allocation, two interactive knowledge-check quizzes, and a critical reflection on how this analysis reshaped my thinking about ethical AI development.

Objective

To critically analyze a real-world AI ethics incident through the lens of three distinct ethical issues, demonstrate how critical thinking about AI bias evolved during the analysis, and present findings in an interactive format that makes the implications tangible for peers working with AI systems in their own organizations.

Process

I selected the Optum healthcare algorithm case from the AI Incident Database, then conducted research using the original Science publication, the New England Journal of Medicine review of race correction in clinical algorithms, and ACLU reporting on algorithmic healthcare decisions. I identified three ethical issues, analyzed their implications with specific data points, and proposed concrete solutions for each. I then transformed the written report into an interactive web experience with visual diagrams, tabbed navigation, and quizzes that reinforce the key concepts — particularly the distinction between aggregate accuracy and equitable accuracy.

Tools & Technologies Used

HTML5, CSS3 (custom properties, responsive grid, gradient backgrounds), JavaScript (DOM manipulation, quiz state management), Claude AI (research assistance and content refinement), Google Fonts.

Value Proposition

Unique Value: This artifact transforms a written ethics assignment into an interactive learning experience. The bias pipeline diagram makes the mechanism of algorithmic discrimination visually intuitive, and the knowledge-check quizzes turn passive reading into active engagement. The case study connects healthcare AI bias to supply chain AI, showing that the same principles — question your proxy variables, audit across demographics, monitor at scale — apply across industries.

Relevance: For peers in the AIML program, this serves as both a reference case study and a practical framework for ethical AI evaluation. The core lesson — that a model can be technically accurate and deeply unjust simultaneously — is essential knowledge for anyone building or deploying AI systems that affect people's lives, whether in healthcare, retail, finance, or any other domain.

References

Obermeyer, Z., Powers, B., Vogeli, C., & Mullainathan, S. (2019). Dissecting racial bias in an algorithm used to manage the health of populations. Science, 366(6464), 447–453.

Vyas, D. A., Eisenstein, L. G., & Jones, D. S. (2020). Hidden in plain sight: Reconsidering the use of race correction in clinical algorithms. New England Journal of Medicine, 383(9), 874–882.

ACLU. (2021). Algorithms are making decisions about health care, which may only worsen medical racism.
